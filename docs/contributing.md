# Guide de Contribution - SlayBot

Merci de votre intérêt pour contribuer à SlayBot ! Cette page résume le processus de contribution.

Pour les détails complets, consultez [CONTRIBUTING.md](https://github.com/ismael-belghazi/Slaybot/blob/main/CONTRIBUTING.md) à la racine du dépôt.

## Comment contribuer

### 1. Fork et cloner

```bash
git clone https://github.com/votre-username/Slaybot.git
cd Slaybot
```

### 2. Créer une branche de feature

```bash
git checkout -b feature/ma-fonctionnalite
```

### 3. Faire vos changements

- Modifiez le code pertinent.
- Ajoutez ou mettez à jour les tests si nécessaire.
- Mettez à jour la documentation.

### 4. Commit et push

```bash
git add .
git commit -m "Ajouter: description de ma contribution"
git push origin feature/ma-fonctionnalite
```

### 5. Créer une Pull Request

- Décrivez clairement vos changements.
- Référencez les issues liées.
- Attendez la review de l'équipe.

## Standards de code

- **Python** : Suivez [PEP 8](https://pep8.org/).
- **Docstrings** : Documentez toutes les fonctions et classes publiques.
- **Tests** : Écrivez des tests pour les nouvelles fonctionnalités.
- **Commits** : Messages descriptifs et cohérents.
- **Documentation** : Mettez à jour les fichiers `.md` pertinents.

## Processus de review

1. Un responsable review votre PR.
2. Des changements peuvent être demandés.
3. Une fois approuvée, elle est fusionnée dans `main`.
4. La documentation est déployée automatiquement sur GitHub Pages.

## Normes de conduite

Tous les contributeurs doivent respecter le [Code de Conduite](code_of_conduct.md).

Comportements attendus :
- Respect mutuel et inclusivité.
- Collaboration constructive.
- Communication honnête et transparente.

## Types de contributions acceptées

- **Bug fixes** : Corrections de bugs existants.
- **Nouvelles fonctionnalités** : Améliorations du système.
- **Documentation** : Clarification ou amélioration des docs.
- **Tests** : Augmentation de la couverture de tests.
- **Issues** : Signalement de bugs ou demandes de fonctionnalités.
